# Edition Matrix (W11)

本文档定义 Community Edition (CE) 与 Commercial Edition (EE) 的功能分层，作为产品、研发与销售对齐基线。

## 能力矩阵

| 模块 | CE | EE |
| --- | --- | --- |
| EVM 登录与会话 | ✅ | ✅ |
| Solana 登录入口（灰度） | ✅（Beta） | ✅ |
| DID 解析（did:ethr） | ✅ | ✅ |
| 基础租户应用配置（域名、回调、链策略） | ✅ | ✅ |
| SDK JS/React 基础能力 | ✅ | ✅ |
| 多租户品牌化（Logo/主题/文案） | ❌ | ✅ |
| 企业级 RBAC（角色/权限模板） | ❌ | ✅ |
| VC 签发与验证策略 | ❌ | ✅ |
| 审计增强（导出、检索、保留策略） | ❌ | ✅ |
| SLA 能力（告警、限流模板、支持通道） | ❌ | ✅ |
| 商业技术支持与升级保障 | ❌ | ✅ |

## 插件化边界（实现约束）

- CE 提供扩展接口：
  - `tenant_branding` 扩展点（品牌资源与主题配置）
  - `tenant_keys` 扩展点（密钥管理策略）
  - `audit_provider` 扩展点（审计事件下沉）
  - `vc_provider` 扩展点（凭证签发与验证）
- EE 通过插件注册方式加载，不直接改写 CE 业务主流程。
- 任何 EE 相关字段必须是可选字段，缺失时 CE 逻辑可独立运行。

## API 暴露策略

- 公共 API（CE/EE 通用）保持稳定：
  - `/auth/*`
  - `/social/*`
  - `/tenant/apps*`（基础能力）
- EE 增强 API 使用独立命名空间：
  - `/enterprise/*`（示例）
  - `/tenant/*/advanced/*`（示例）
- 版本策略：向后兼容优先，破坏性变更走主版本升级。

## 交付与部署建议

- CE：
  - 公开仓库源码 + 文档 + 最小部署脚本
  - 适配本地开发与社区自部署
- EE：
  - 私有包或私有镜像发布
  - License Key 或授权网关校验
  - 企业支持渠道与 SLA 响应承诺

## 变更流程

- 新增能力前先标注归属：`CE` / `EE` / `Both`。
- 若能力归属 EE，但存在通用价值，先抽象扩展点后再实现。
- PR 模板应增加“Edition Impact”勾选项，避免跨边界耦合。
