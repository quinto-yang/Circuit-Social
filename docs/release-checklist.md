# Release Checklist (W12)

目标：形成可重复的发布流程，覆盖构建、测试、文档、版本与回滚准备。

## 1) 版本与变更冻结

- [ ] 确认发布版本号（遵循 semver）。
- [ ] 冻结高风险功能变更，仅接收发布阻断级修复。
- [ ] 汇总本次变更说明（功能、修复、迁移影响）。

## 2) 代码质量门禁

- [ ] 一键门禁脚本通过：
  - `npm run release:gate`（建议默认包含 demo e2e）
- [ ] API 构建与测试通过：
  - `npm run test:api`
- [ ] 全量类型检查通过：
  - `npm run typecheck`
- [ ] Web smoke + mobile-smoke 通过：
  - `npm run e2e:run`
- [ ] 如涉及钱包登录链路，`npm run test:wallet` 通过并附运行记录。

## 3) 平台化专项检查（12 周路线收口）

- [ ] 多链开关行为正确（Solana flag 开关、禁用态提示、错误文案）。
- [ ] DID 状态在 `/me`、`/profile`、认证响应中一致。
- [ ] SDK（`@cx/sdk-js`、`@cx/sdk-react`）可构建并可在 demo 中运行。
- [ ] SaaS 租户基础能力可用（创建应用、域名白名单、应用配置编辑/筛选、品牌配置、密钥轮换）。
- [ ] CE/EE 文档边界更新完成（`licensing` + `edition-matrix`）。

## 4) 文档与对外信息

- [ ] README 命令与环境变量说明与当前实现一致。
- [ ] 错误码文档与后端 source-of-truth 一致。
- [ ] 新增能力有最小使用说明或截图（如管理后台变更）。
- [ ] 风险项和已知限制写入发布说明（例如 Solana 开关默认关闭、密钥轮换冷却时间策略）。

## 5) 配置与环境检查

- [ ] 生产环境变量核对（API origin、功能开关、cookie 相关配置）。
- [ ] CORS 白名单已按生产域名收敛。
- [ ] 上传存储策略确认（本地盘 or S3/R2）并符合发布环境预期。
- [ ] 日志级别与敏感字段脱敏策略已确认。

## 6) 数据与兼容性

- [ ] 若涉及数据结构变更，提供迁移/回滚说明。
- [ ] 前后端 API 兼容性检查完成（新增字段为可选、旧客户端可运行）。
- [ ] 关键用户流程回归：登录、发消息、群聊管理、朋友圈、资料更新。

## 7) 发布执行

- [ ] 在发布分支打 tag（例如 `v0.x.y`）。
- [ ] 生成发布记录（Release Notes）。
- [ ] 部署顺序确认：API -> Web -> 验证。
- [ ] 发布后 30 分钟观测：错误率、接口耗时、登录成功率。

## 8) 回滚预案

- [ ] 保留上一稳定版本镜像/tag。
- [ ] 明确一键回滚命令或平台操作路径。
- [ ] 明确回滚触发阈值（错误率、核心链路不可用时长）。
- [ ] 指定回滚负责人与通知渠道。

## 9) 发布负责人签字（模板）

- 发布负责人：
- 测试负责人：
- 安全审计负责人：
- 发布时间：
- 回滚预案确认：已确认 / 未确认
