# Security Checklist (W12)

目标：在发布前完成最小可执行安全审计，覆盖认证、防重放、租户隔离、上传与会话安全。

## 1) 认证与签名链路

- [ ] `POST /api/auth/nonce` 必须校验地址格式、链类型与支持链范围。
- [ ] nonce 必须一次性消费，重复验签应返回失败（防重放）。
- [ ] nonce 必须有 TTL，过期后验签明确失败。
- [ ] `auth/verify` 必须校验签名地址与声明地址一致。
- [ ] 不同链类型（`evm` / `solana`）必须走各自 payload 规则，不可混用。
- [ ] 登录失败返回结构化错误码，避免泄漏服务内部细节。

## 2) Session 与 Cookie

- [ ] session cookie 启用 `httpOnly`。
- [ ] session cookie 在生产环境启用 `secure`。
- [ ] session cookie 设置合理 `sameSite` 策略（默认 `lax`，跨站场景明确评估）。
- [ ] 登出后 session 必须立即失效，`/api/me` 返回未登录态。

## 3) 多租户隔离

- [ ] 租户应用查询仅返回当前登录用户拥有的数据。
- [ ] 租户应用更新/域名管理必须校验 owner 身份，禁止越权访问。
- [ ] 域名白名单必须规范化（小写、去空白）并防重复。
- [ ] 回调地址必须校验协议（仅 `http/https`）与格式有效性。
- [ ] 若已配置域名白名单，回调地址 host 必须在白名单内（本地调试域名可例外）。
- [ ] 禁止将他人 appId 通过参数注入进行修改。
- [ ] 租户密钥轮换存在冷却策略，防止高频误操作/滥用。

## 4) 上传与内容安全

- [ ] 上传接口限制 MIME 与扩展名，仅允许图片。
- [ ] 上传大小限制生效（当前 3MB）。
- [ ] 上传路径不可被用户输入直接拼接，避免路径穿越。
- [ ] 富文本/消息内容展示时做最小转义策略，避免 XSS 注入。

## 5) API 与错误响应

- [ ] 统一错误 JSON 响应结构（`ok/error/errorCode/path/statusCode`）保持一致。
- [ ] 未知异常不返回堆栈到客户端。
- [ ] 关键安全事件（登录失败、越权访问、租户配置变更）纳入审计日志。

## 6) CORS 与跨域

- [ ] 明确允许来源列表，避免生产环境 `*`。
- [ ] 仅暴露必要 headers 与 methods。
- [ ] 带凭证跨域时验证 `origin + credentials` 组合配置。

## 7) E2EE 与密钥相关（当前能力范围）

- [ ] 客户端仅上传公钥，不上传私钥。
- [ ] 会话列表最后一条消息展示需解密后回显（已支持）。
- [ ] 对未启用加密成员的降级提示必须是会话级而非全局级（已支持）。
- [ ] 评估本地密钥存储风险（`localStorage`）并记录后续硬化计划。

## 8) 执行命令（发布前必跑）

```bash
npm run test:api
npm run typecheck
npm run e2e:run
```

如涉及钱包真链路回归，追加：

```bash
npm run test:wallet
```

## 9) 审计结论模板

- 风险级别：`Critical / High / Medium / Low`
- 影响范围：认证 / 会话 / 租户 / 内容 / 前端
- 复现步骤：最小复现路径
- 修复状态：未修复 / 已修复 / 已缓解
- 发布阻断：是 / 否
