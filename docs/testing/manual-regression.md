# CX 社交 H5 手工回归清单

## 执行方式
- 命令：`npm run test:manual`
- 目标环境：桌面 Chrome、iOS Safari、Android Chrome、MetaMask Browser、OKX Wallet Browser。
- 每次回归前建议先重置测试环境，并确认前后端都运行在本地测试地址。

## 核心链路
1. 钱包登录
   - 打开登录页，确认有“测试钱包/小额地址”安全提示。
   - 使用 MetaMask 或 OKX Wallet 登录并签名。
   - 预期：进入聊天页，能看到欢迎私聊和官方群。
2. 会话与返回
   - 打开 `Circuit Concierge`，点击左上角“返回”。
   - 预期：能回到会话列表，无卡死、无遮罩拦截。
3. 消息发送去重
   - 在欢迎私聊连续发送 2 条不同文本，刷新页面后再次查看。
   - 预期：每条消息只保留 1 份，不出现重复。
4. 群聊管理
   - 打开 `Circuit Lounge`，展开群管理抽屉，检查群号展示。
   - 使用群主账号执行禁言/解禁/移出成员。
   - 预期：群号可见；禁言立即生效；解禁后成员恢复发言。
5. 好友链路
   - 通过用户 ID、钱包地址各发起一次好友申请。
   - 另一账号接受后，检查是否生成私聊。
   - 预期：申请成功；重复申请有明确提示；接受后私聊可复用。
6. 朋友圈
   - 发布文本动态、图文动态；再用另一账号举报动态。
   - 预期：动态立即出现在列表；图片正常展示；举报反馈成功。
7. 资料编辑
   - 修改昵称、简介、头像 URL、主钱包。
   - 预期：保存后“我的”页立即展示最新信息。

## 移动端专项
- 检查输入框、底部 Tab 和抽屉在 iPhone 与 Android 小屏下是否遮挡。
- 检查钱包内置浏览器里签名弹窗返回后，页面状态是否丢失。
- 检查图片上传预览、长昵称、长钱包地址是否溢出。

## 缺陷记录建议
- 记录环境：设备、浏览器/钱包版本、构建 commit。
- 记录输入：账号、钱包地址、群号、动态内容。
- 记录结果：预期、实际、截图/录屏、复现概率。
